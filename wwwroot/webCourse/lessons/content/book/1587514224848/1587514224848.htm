一、   内置类类型public class System.Object是一切类型树型继承拓展的绝对根（内置九个方法，零个属性。没有履行拓展接口）。

根据类型定义源码public class System.Object，基于图 5‑18、表 5‑2、表 5‑3，可挖掘出类型的System.Object名称空间（VS对象浏览器中可查看到处于mscorlib.dll程序集文件），属于类类型（未履行拓展任何接口）、而且是引用类型，public到访性，非sealed，非abstact、非static。含有如下属性、方法。

l  public Object()。本类型的构造方法。在此为public到访性，非virtual、非abstact、非static、没有返回对象的类型声明也没有void关键词（构造方法特殊于通常方法之处)、方法名称Object()（构造方法名称必须和类名称完全一致）、没有定义参数。类型对象化时（例如，new关键词创建对象时），构造方法会被自动调用。还将在表 5‑3深入。

l  ~Object()。本类型的析构方法。在此无到访性（析构方法特殊于通常方法之处），非virtual、非abstact、非static、没有返回对象的类型声明也没有void关键词（析构方法特殊于通常方法之处)、方法名称Object()（析构方法名称必须是类名加~前缀）、没有定义参数。类型的对象销毁时（例如，.Net Core平台自动回收对象垃圾时），析构方法会被自动调用，还将在表 5‑3深入。

l  public virtual System.Int32 GetHashCode()。返回当前对象的哈希源码/摘要源码。类型里拥有GetHashCode()方法（返回Hash）的真正目的是，允许该类型在内部使用HashTable的集合中可以作为Key，因为HashTable需要这些哈希码。例如Dictionary<TK, TV>。为了让HashTable可以正确的工作，Hash码有一个要求：如果两个实例被认为是相等的，那么它们必须返回相同的hash码。如果没有实现这个要求，那么你可能会发现这个类型作为Dictionary的Key的时候，会有一些意想不到的结果。所以如果重写了object.Equals()方法，那么就得重写object.GetHashCode()方法。相同的对象一定返回相同的哈希源码，不同的对象不一定返回不同的哈希源码（当然这种可能性很小）,即，若两对象Equals()相等，那么GetHashCode()必定相等；但是反过来，若两对象GetHashCode()相等，不一定Equals()相等。此外，当把对象添加到一个键值对这样的容器（如Dictionary<Key,Value>）时，可以调用对象的这个函数来得到一个唯一的哈希源码，把该值放入键值对容器中作为唯一的键值，从而查增改删。总之，GetHashCode()功能与性能陷进比较多，需要在实际源码中具体思考应对。返回类似GUID的全球唯一值需要通过.Net Core内置的System.Guid类型实现。

这是Object类关于这两个方法的源码，可以看出，Object类默认的equals比较规则就是比较两个对象的内存地址。而hashcode是本地方法，java的内存是安全的，因此无法根据散列码得到对象的内存地址，但实际上，hashcode是根据对象的内存地址经哈希算法得来的。

假如只重写equals而不重写hashcode，那么Student类的hashcode方法就是Object默认的hashcode方法，由于默认的hashcode方法是根据对象的内存地址经哈希算法得来的，显然此时s1!=s2,故两者的hashcode不一定相等。

然而重写了equals，且s1.equals(s2)返回true，根据hashcode的规则，两个对象相等其哈希值一定相等，所以矛盾就产生了，因此重写equals一定要重写hashcode，而且从Student类重写后的hashcode方法中可以看出，重写后返回的新的哈希值与Student的两个属性有关。

两个对象相等，hashcode一定相等、两个对象不等，hashcode不一定不等、hashcode相等，两个对象不一定相等、hashcode不等，两个对象一定不等。

l  public static System.Boolean ReferenceEquals(System.Object objA, System.Object objB) [[ii]]。objA对象、objB对象引用相等，则返回true，否则返回false。static声明。

l  public static System.Boolean Equals(System.Object objA, System.Object objB) [[iii]]。objA对象、objB对象相等（引用相等或拷贝相等），则返回true，否则返回false。static声明。

l  public virtual System.Boolean Equals(System.Object obj)[[iv]]。当前对象、obj对象相等（引用相等或拷贝相等），则返回true，否则返回false。virtual声明。

l  public System.Type GetType()。返回当前对象的类型的对象（在此即System.Object）。

l  protected System.Object MemberwiseClone()。返回当前对象的属性方法这两种成员的克隆的对象（克隆成员，而不克隆成员所引用的对象，又称为“浅克隆”。克隆成员，而又克隆成员所引用的对象，则称为“深克隆”，可以通过对象序列化/对象反序列化实现深克隆，将在5.7.6 节详述。另外，System.ICloneable接口也同时提供了浅克隆、深克隆的功能，但业界普遍反映不可预测结果的现象比较普遍，尽量避免使用）。

l  public virtual System.String ToString()。返回当前对象的字符串对象（在此即“System.Object”）。virtual声明。
